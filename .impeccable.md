# Design context and guidelines

## Design Context

### Users
Operators and developers who need to monitor URL uptime. They use the app in a professional context (often in browser tabs or after hours) and want quick status visibility and reliable checks.

### Brand Personality
Clear, reliable, minimal. Voice is neutral and functional. Emotional goal: confidence that the system is under control.

### Aesthetic Direction
Clean and readable. Prefer restraint over decoration. Support both light and dark themes with a user-controlled toggle; default to system preference (prefers-color-scheme) when no preference is saved. Avoid generic purple/gradient AI aesthetics. Use semantic status colors (e.g. green/red for up/down) that work in both themes.

### Design Principles
1. **Clarity** – Status and configuration should be scannable at a glance.
2. **Consistency** – Use a single set of design tokens (page background, card background, text primary/muted, borders, accents) across all pages.
3. **Theme parity** – Light and dark modes are first-class; every surface uses tokens so both themes look intentional.
4. **Accessibility** – Sufficient contrast, no reliance on color alone for status where possible, focus states on interactive elements.
5. **No !important** – Prefer cascade and token-based overrides.
